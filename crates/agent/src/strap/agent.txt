### agent — Self-Management, Cognition & Agent Registry
USE THIS when: spawning sub-agents, tracking multi-step work, searching memory, managing sessions, analyzing images, asking the user a question, OR delegating to/managing installed agents.
CRITICAL: To delegate to a NAMED agent (e.g. "chief-of-staff"), use agent(resource: "registry", action: "delegate") — NOT agent(resource: "task", action: "spawn"). Delegation loads the agent's full identity, plugins, and skills. Spawning creates a blank sub-agent with no personality.

**Sub-agents (parallel work):**
Spawn sub-agents for independent work. Two modes:
- **wait: true (DEFAULT)** — You block until the sub-agent finishes. Result returns inline. Use this for most cases.
- **wait: false** — Sub-agent runs in background. Its result is automatically delivered to your conversation when it completes. Use this only when you have other work to do in parallel.
- agent(resource: task, action: spawn, prompt: "Research competitor pricing", tools: ["web"]) — Spawn with web tool docs
- agent(resource: task, action: spawn, prompt: "Check inbox for urgent items", plugins: ["PLUG-PJ3Z-ECFV"], tools: ["loop", "message"]) — Spawn with plugin + tools
- agent(resource: task, action: spawn, prompt: "...", wait: false, tools: ["web"]) — Background with tool docs
- agent(resource: task, action: status, agent_id: "...") — Check status of a background agent
- agent(resource: task, action: cancel, agent_id: "...") — Cancel a running sub-agent

**Batch parallel spawn (preferred for 2+ independent tasks):**
- agent(resource: task, action: spawn_parallel, tasks: [{prompt: "Search site A", tools: ["web"]}, {prompt: "Search site B", tools: ["web"]}])
- All tasks run concurrently, results collected and returned together
- Each sub-agent gets its own browser session — no tab conflicts

**Searching with an explore sub-agent (agent_type: "explore"):**
- agent(resource: task, action: spawn, prompt: "Find where X is configured across the repo", agent_type: "explore") — a fast, READ-ONLY search specialist that issues parallel greps/reads and reports back. Keeps bulky search output out of your context.
- Delegate the search when you're hunting for code/files/answers and aren't confident you'll find the match in the first couple of tries. When you already know the exact file path, just read it directly — don't spawn an agent for that.

**CRITICAL: Sub-agents are born blind.** They only know what you pass them.
- **plugins**: Pass plugin install codes the sub-agent needs. It gets the plugin's description, skills, and usage docs.
- **tools**: Pass STRAP tool names ("web", "loop", "message", "system", "agent", "skill", "event", "mcp"). It gets the full tool documentation.
- **skills**: Pass skill names for SKILL.md instructions pre-loaded into context.
If you use plugins or specific tools, you MUST pass them to any sub-agent that will need them too.

**Work tracking (keep yourself on task):**
For multi-step work, create tasks to track your progress. This prevents you from losing focus or repeating steps.
- agent(resource: task, action: create, subject: "Test shell tool") — Create a trackable step
- agent(resource: task, action: update, task_id: "1", status: "in_progress") — Mark a task in_progress BEFORE you begin it; keep only ONE task in_progress at a time
- agent(resource: task, action: update, task_id: "1", status: "completed") — Mark a task completed as soon as you finish it; don't batch completions (pending → in_progress → completed)
- agent(resource: task, action: list) — See all tasks and sub-agents
- agent(resource: task, action: delete, task_id: "1") — Remove a task
Only mark a task completed when it is FULLY accomplished. If you hit an error, a blocker, or only partly finished, keep it in_progress and create a follow-up task — never mark incomplete or failed work as completed.

When to spawn vs do it yourself:
- **ALWAYS spawn** when: comparing across multiple websites/sources, researching multiple independent topics, any task where 2+ web searches could run simultaneously
- Spawn when: multiple independent tasks, long-running research, tasks that don't depend on each other
- Do it yourself when: simple single task, tasks that depend on each other's results, quick lookups

**Memory (3-tier persistence):**
- agent(resource: memory, action: store, key: "user/name", value: "Alice", layer: "tacit") — Store a fact
- agent(resource: memory, action: recall, key: "user/name") — Recall a specific fact
- agent(resource: memory, action: search, query: "...") — Search across all memories
- agent(resource: memory, action: list) — List stored memories
- agent(resource: memory, action: delete, key: "...") — Delete a memory
- agent(resource: memory, action: clear) — Clear all memories
Layers: "tacit" (long-term preferences — MOST COMMON), "daily" (today's facts, auto-expires), "entity" (people/places/things)

**Registry (installed agent management + delegation):**
When user mentions a named agent or wants to list/activate/delegate to installed agents.
- agent(resource: registry, action: delegate, name: "chief-of-staff", prompt: "Check my email for urgent items") — Delegate to a NAMED agent
- agent(resource: registry, action: delegate, id: "13223420-...", prompt: "...", wait: false) — Background delegation
- agent(resource: registry, action: list) — List available agents
- agent(resource: registry, action: activate, name: "chief-of-staff") — Activate an agent
- agent(resource: registry, action: deactivate) — Deactivate current agent
- agent(resource: registry, action: info, name: "chief-of-staff") — Show agent details
- agent(resource: registry, action: create, name: "my-agent", agent_md: "# Agent...") — Create a user agent
- agent(resource: registry, action: install, code: "AGNT-XXXX-XXXX") — Install from marketplace
