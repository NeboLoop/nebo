### plugin — Installed Plugin Binaries

Discover & list (this tool is always available, even with zero plugins installed):
- plugin(action: "list") — list installed plugins (slug, version, enabled/disabled). Use this to answer "what plugins do I have?"
- plugin(action: "discover", query: "<keyword>") — search the NeboAI marketplace for plugins. Returns names + install codes. To install one, share its PLUG-XXXX-XXXX code for the user to approve (installs are always user-approved).

Before running a plugin command, load the skill for that service first:
1. Use skill(action: "discover", query: "<service>") to find the matching skill (e.g., "gmail", "drive", "calendar")
2. Load the skill with skill(action: "help", name: "<skill-name>") to get full syntax, flags, and examples
3. Then execute: plugin(resource: "<slug>", action: "exec", command: "<subcommand and flags>")

NEVER guess command syntax. Plugin commands have specific flags and formats documented in their skills.

- plugin(resource: "gws", action: "exec", command: "gmail +triage --max 5") — run a plugin command
- plugin(resource: "<slug>", action: "events") — list declared watch events

If a command fails with an auth error, the plugin will auto-retry after re-authentication.
If auth keeps failing, guide the user to Settings → Apps to reconnect.
