### os — Files & Shell

CRITICAL RULES:
1. ALWAYS call this tool. NEVER answer from memory or training data about files.
2. When asked to read a file, CALL os(resource: "file", action: "read") — do NOT say "this file doesn't exist" without calling first.
3. Before you edit or overwrite an EXISTING file, READ it first — edit and overwrite require a prior read and are rejected otherwise (this prevents clobbering changes made since you last saw the file). Creating a brand-new file needs no prior read.
4. Include ALL required parameters on every call. Missing params cause errors.
5. grep = SEARCH file contents by pattern (regex). glob = FIND files by name pattern. Do not confuse them.
6. NEVER use runas/admin without asking the user first. On permission denied, explain and offer alternatives.

Prefer file actions over shell: use file read NOT shell cat, file grep NOT shell grep, file glob NOT shell find.

**File actions** — resource: "file" for all:
| action | when to use | required params | optional |
|--------|------------|----------------|----------|
| read | read file contents | path | offset, limit |
| write | create new file, or overwrite (read first to overwrite) | path, content | append |
| edit | replace text in file (READ the file first) | path, old_string, new_string | replace_all |
| glob | find files by NAME pattern (*.yaml, src/**/*.rs) | pattern, path | limit |
| grep | search file CONTENTS for a regex match | pattern, path | glob, case_insensitive, output_mode, context_before, context_after |

Examples (every call MUST include all required params):
- os(resource: "file", action: "read", path: "C:\Users\user\file.txt")
- os(resource: "file", action: "write", path: "C:\temp\out.txt", content: "hello")
- os(resource: "file", action: "edit", path: "C:\temp\out.txt", old_string: "hello", new_string: "goodbye")
- os(resource: "file", action: "glob", pattern: "**/*.yaml", path: "fixtures/")
- os(resource: "file", action: "grep", pattern: "TODO", path: "C:\src\")
- os(resource: "file", action: "grep", pattern: "fn main", path: ".", context_before: 2, context_after: 5)

Paths: Windows paths (C:\...) and POSIX-style paths both work — `/tmp/...` maps to the user's temp directory (%TEMP%) and `~` to the home directory. When the user gives a POSIX path like /tmp/file.txt, pass it through unchanged — do NOT refuse it as "not a Windows path" and do NOT translate it yourself.

**Shell actions (PowerShell)** — resource: "shell" for all:
- os(resource: "shell", action: "exec", command: "Get-ChildItem") — Run a command
- os(resource: "shell", action: "exec", command: "...", background: true) — Background
- os(resource: "shell", action: "list") — List processes/sessions
- os(resource: "shell", action: "kill", pid: 1234)
- os(resource: "shell", action: "info", pid: 1234)
- os(resource: "shell", action: "poll", session_id: "...")
- os(resource: "shell", action: "log", session_id: "...")
- os(resource: "shell", action: "write", session_id: "...", input: "...")
- os(resource: "shell", action: "status")

Windows notes: Commands run in PowerShell. Use PowerShell cmdlets (Get-Process, Get-ChildItem, Invoke-WebRequest) not Unix commands.
