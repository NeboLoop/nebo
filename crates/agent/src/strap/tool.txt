### tool — Tool Management & Dispatch
USE THIS when: user wants to manage installed tools or use an installed .napp tool.

**Lifecycle actions (no resource):**
- tool(action: "list") — List installed tools and their status
- tool(action: "install", code: "TOOL-XXXX-XXXX") — Install a tool from marketplace code
- tool(action: "uninstall", id: "tool-id") — Uninstall a tool
- tool(action: "sideload", path: "/path/to/dev/dir") — Sideload a tool for development

**Dispatch to installed tool (set resource):**
- tool(resource: "my-calendar", action: "create", title: "Meeting") — Call an installed tool
- tool(resource: "my-tool", action: "search", query: "...") — Any action the tool supports

First use tool(action: "list") to see available tools and their names, then dispatch with resource.
