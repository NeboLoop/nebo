## Appendix D: Known open items

1. **Multi-turn CLI sessions** render prior tool results as `[Tool result unavailable]` in the rebuilt prompt (cli.rs `build_prompt_from_messages`) — long sessions lose tool context with CLI providers. Not Windows-specific. Surfaced during AT runs.
2. **`message(notify)` DND**: status unimplemented on Windows (returns "unavailable"); on/off toggles unimplemented everywhere (plan AT-23 is aspirational).
3. **`nebo-cli chat` one-shot** bypasses tools/routing entirely (debug utility) — the integration plan's `nebo chat` commands cannot exercise the real pipeline from the CLI.
4. **macOS verification**: all Windows-gated changes are cfg'd out on macOS; `nebo-types` cross-checks clean for `aarch64-apple-darwin` (full workspace cross-check impossible from Windows — `ring` needs an Apple toolchain). CI macOS job is the authoritative gate before merge.
