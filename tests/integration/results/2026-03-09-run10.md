## Bugs Found

### NEW in Run 10
1. **role(create) literal \\n bug** — `role_md` parameter writes `\\n` as literal backslash-n instead of newlines. ROLE.md becomes unparseable, role not detected by scanner. Workaround: write ROLE.md manually via file write.

### Persistent (from previous runs)
2. **AT-17 sub-agent spawn** — "No messages in session" error
3. **Marketplace install 404** — All 3 code types (SKILL/WORK/ROLE) return 404 "invalid code". Was returning success-but-no-persist in run 08, now fully fails.
4. **AT-21 role description** — Shows "-" when created via role(create) tool (due to \\n bug corrupting frontmatter)
5. **S-05 trigger display** — Triggers in skill frontmatter not shown in catalog listing

## Publisher's Guide Discrepancies
1. **Fallback** — Publisher's guide says fallback can reference another activity. Implementation uses enum: `notify_owner`, `skip`, `abort`
2. **Tools restriction** — Publisher's guide says `tools` field. Implementation uses `skills` field on Activity
3. **role.json** — Not auto-generated by `role(create)` unless `role_json` parameter is explicitly provided
4. **Triggers in info** — `role(info)` doesn't display trigger config from role.json
